# AI 반응형 몬스터 설계 (Red Shadow / 붉은 그림자)

> 핵심 비밀 컨셉: **적은 "하드코딩(스크립트) 두뇌"와 "반응형 AI 두뇌"를 동시에 갖는다.**
> 난이도가 둘의 **비율(ai_ratio)** 을 조절하고, AI가 강해져도 **의도적으로 약점(텔·빈틈)** 을 남겨
> 플레이어가 패턴을 학습·공략할 수 있게 한다.

---

## 1. 설계 원칙

1. **블렌딩, 100% AI 아님** — 매 결정마다 확률적으로 스크립트 두뇌 / AI 두뇌 중 하나가 행동을 결정.
2. **의도적 약점** — 난이도가 최고여도 절대 사라지지 않는 학습 가능한 규칙(고정 텔레그래프, 공격 후 경직, 사각지대)을 유지.
3. **몸과 두뇌 분리** — `Enemy.gd`(몸/물리/HP)와 `EnemyBrain`(판단)을 분리해 적마다 재사용·교체 가능.
4. **고양이 컨셉 연결** — 스텔스/블링크/대시로 인지를 흔들고 사각을 파고드는 플레이가 보상받도록.

---

## 2. 모듈 구조

```
Enemy (CharacterBody2D)         # 몸: 이동, HP, 넉백, 피격, 텔레그래프 재생
 └─ EnemyBrain (Node)           # 판단: 매 틱 정책 선택 → 행동 명령 반환
      ├─ ScriptedPolicy         # 정해진 패턴 (빈틈 내장)
      ├─ AIPolicy               # 반응형 (Perception 기반 결정)
      └─ Perception             # 플레이어 상태 인지(거리/방향/행동/높이차/스텔스)
```

### 결정 루프 (의사 코드)
```gdscript
# EnemyBrain, decision_interval 마다 호출
func decide(perception) -> Action:
    if randf() < ai_ratio:
        return ai_policy.decide(perception)     # 반응형
    else:
        return scripted_policy.decide(state)    # 정해진 패턴
```

### 행동 명령 (Action)
- `IDLE` / `PATROL` / `APPROACH` / `RETREAT`
- `TELEGRAPH(attack_type)` → `ATTACK(attack_type)` (텔레그래프 필수 경유)
- `DODGE` (AI 전용, 어려움에서만 등장)

---

## 3. 난이도 = 비율 다이얼

`Enemy`/`EnemyBrain`의 `@export` 변수로 노출, 인스펙터에서 실시간 조정.

| 파라미터 | 의미 | 쉬움 | 보통 | 어려움 |
|---|---|---|---|---|
| `ai_ratio` | AI가 결정을 가져갈 확률 (0~1) | 0.2 | 0.5 | 0.85 |
| `reaction_delay` | AI 반응 지연(초) | 0.50 | 0.30 | 0.15 |
| `tell_time` | 공격 전 텔레그래프 길이(초) — **약점** | 0.45 | 0.35 | 0.25 |
| `decision_interval` | 판단 주기(초) | 0.40 | 0.28 | 0.18 |
| `perception_range` | 인지 반경(px) | 280 | 360 | 460 |
| `post_attack_stun` | 공격 후 고정 경직(초) — **약점** | 0.40 | 0.30 | 0.22 |

> 약점 파라미터(`tell_time`, `post_attack_stun`)는 **0 이하로 내려가지 않도록 하한선**을 코드로 강제한다.

---

## 4. 의도적 약점 규칙 (절대 불변)

1. **텔레그래프 보장** — 모든 공격은 `tell_time` 동안 예비동작(번쩍임/자세)을 반드시 재생. 반응 회피 가능.
2. **공격 후 경직** — 공격 직후 `post_attack_stun` 동안 무방비. 반격 타이밍.
3. **사각지대** — 등 뒤 / 머리 위 일정 각도는 인지율 급감. 블링크·대시로 파고들기 보상.
4. **스텔스 페널티** — 플레이어가 스텔스 중이면 `perception_range`와 인지율이 크게 감소.

---

## 5. 인지 모델 (Perception)

AI가 매 틱 읽는 정보:
- `distance` : 플레이어와의 거리
- `dir_to_player` : 방향(좌/우, 높이차)
- `player_action` : 최근 행동(공격/블링크/대시/스텔스 여부)
- `in_blind_spot` : 사각지대 여부
- `player_stealthed` : 스텔스 상태

AI 정책은 이 정보로 반응:
- 플레이어가 공격을 시작 → 거리에 따라 `RETREAT` 또는 `DODGE`
- 플레이어가 경직/착지 빈틈 → `APPROACH` 후 `TELEGRAPH→ATTACK`
- 사각/스텔스 → 인지 실패 → `PATROL` 유지 (공략 포인트)

## 6. 구현 단계 (로드맵)

- [ ] **1단계** `Enemy.gd`에 상태머신 + 텔레그래프/경직 재생 추가 (몸)
- [ ] **2단계** `EnemyBrain` 노드 + `Action` 열거형 + 결정 루프
- [ ] **3단계** `Perception` 헬퍼 (플레이어 참조, 거리/사각/스텔스 계산)
- [ ] **4단계** `ScriptedPolicy` — 빈틈 있는 고정 패턴 1~2종
- [ ] **5단계** `AIPolicy` — 반응형 회피/접근/반격
- [ ] **6단계** 난이도 프리셋(쉬움/보통/어려움) `@export` 묶음 + 하한선 강제
- [ ] **7단계** 디버그 오버레이(현재 정책·인지상태 표시)로 비율 체감 튜닝

---

## 7. 열린 질문 (다음 의논)

- AI 정책 방식: **유틸리티 기반 점수제** vs **간단 행동트리** — 첫 적은 유틸리티 점수제가 튜닝 쉬움.
- 난이도 전환 시점: 스테이지별 고정 vs 적별 개별 설정 vs 동적(플레이어 실력 추적).
- "학습 가능"의 범위: 패턴 고정 vs 약간의 랜덤 시드 — 너무 랜덤이면 학습 불가, 너무 고정이면 단조.
